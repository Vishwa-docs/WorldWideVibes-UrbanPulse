# UrbanPulse Data Sources

## Overview

UrbanPulse ingests data from multiple categories of sources:

1. **Montgomery Open Data Portal** — Official city datasets via 8 ArcGIS REST endpoints
2. **Open-Meteo Weather API** — Real-time weather conditions and 7-day forecast
3. **US Census ACS** — 31 demographic variables (education, commuting, transportation)
4. **Bright Data (4 Products)** — POI, SERP, review, and web scraping signals
5. **Google Places API** — Walk score and nearby amenities
6. **Seed / Sample Data** — 50 realistic demo properties generated by the seed script

---

## Montgomery Open Data Portal

**Portal URL**: https://opendata.montgomeryal.gov

The portal is powered by ArcGIS and provides REST API access. UrbanPulse queries
these endpoints **live** — no CSV download required.

### ArcGIS REST Endpoints (Live Queries)

| Dataset | Endpoint | Description |
|---------|----------|-------------|
| **311 Service Requests** | `mgmgis.montgomeryal.gov/.../Received_311_Service_Request/FeatureServer/0` | Service requests by category, status, location |
| **Business Licenses** | `mgmgis.montgomeryal.gov/.../Business_License/FeatureServer/0` | Active business licenses with company name, type, address |
| **Most Visited Locations** | `services7.arcgis.com/.../Most_Visited_Locations/FeatureServer/0` | Foot traffic data: visits, visitors per location |
| **Visitor Origin** | `services7.arcgis.com/.../Visitors_Origin/FeatureServer/0` | Where visitors come from — origin-destination patterns |
| **Code Violations** | `mgmgis.montgomeryal.gov/.../Code_Violations/FeatureServer/0` | Code enforcement cases — blight/crime proxy |
| **Opportunity Zones** | `services7.arcgis.com/.../Opportunity_Zones/FeatureServer/0` | Federal tax incentive zones for investment |
| **City-Owned Properties** | `mgmgis.montgomeryal.gov/.../City_Owned_Properities/FeatureServer/0` | Parcels owned by the City of Montgomery |
| **Building Permits** | `mgmgis.montgomeryal.gov/.../Construction_Permits/FeatureServer/0` | Construction & renovation permits (economic activity signal) |

### Weather API (Open-Meteo)

| Endpoint | Description |
|----------|-------------|
| `GET /api/weather/current` | Current temperature, humidity, wind, precipitation for Montgomery |
| `GET /api/weather/forecast` | 7-day daily forecast with highs, lows, precipitation |

Source: [Open-Meteo](https://open-meteo.com) — free, no API key required.

**All 8 endpoints return 100% live data — no mock or simulated fallbacks.** If an endpoint is temporarily unreachable, the service logs a warning and returns an empty list.

These eight datasets directly address the hackathon challenge areas:

- **Compare business licenses with foot traffic trends** → Business Licenses + Most Visited Locations + Visitor Origin
- **Identify and address vacant properties** → 311 Service Requests (vacant/blight category) + Code Violations + property `is_vacant` flag
- **Analyze and improve usage of city-owned properties** → City-Owned Properties endpoint + Property `is_city_owned` flag + opportunity scoring
- **Attract new investment** → Opportunity Zones + Building Permits as economic growth signals
- **Safety & blight analysis** → Code Violations as enforcement proxy for crime/blight patterns

## Column Mapping Configuration

The ingestion script uses a column mapping defined in `config/cities/montgomery.json`
to normalize source CSV columns to the UrbanPulse database schema.

### Parcels Mapping

| UrbanPulse Field | Source Column | Description |
|------------------|--------------|-------------|
| `parcel_id` | `PARCEL_ID` | Unique parcel identifier |
| `address` | `FULL_ADDRESS` | Street address |
| `latitude` | `LATITUDE` | GPS latitude |
| `longitude` | `LONGITUDE` | GPS longitude |
| `property_type` | `PROPERTY_TYPE` | residential, commercial, mixed, vacant_land |
| `zoning` | `ZONING` | Zoning code (R-1, C-2, etc.) |
| `is_vacant` | `VACANT` | Y/YES/TRUE/1 = vacant |
| `lot_size_sqft` | `LOT_SIZE` | Lot area in square feet |
| `assessed_value` | `ASSESSED_VALUE` | Tax-assessed value |
| `year_built` | `YEAR_BUILT` | Year structure was built |
| `neighborhood` | `NEIGHBORHOOD` | Neighborhood name |

### Incidents Mapping

The incidents CSV is expected to have these columns (case-sensitive):

| Column | Description |
|--------|-------------|
| `INCIDENT_ID` | Unique incident identifier |
| `INCIDENT_TYPE` | Category (theft, vandalism, assault, etc.) |
| `LATITUDE` | GPS latitude |
| `LONGITUDE` | GPS longitude |
| `NEIGHBORHOOD` | Neighborhood name |
| `SEVERITY` | low, medium, or high |

### City Projects Mapping

| Column | Description |
|--------|-------------|
| `PROJECT_NAME` | Name of the project |
| `PROJECT_TYPE` | streetscape, infrastructure, park, transit |
| `STATUS` | planned, ongoing, completed |
| `LATITUDE` | GPS latitude |
| `LONGITUDE` | GPS longitude |
| `BUDGET` | Project budget |
| `DESCRIPTION` | Textual description |

### Customizing Mappings

To adapt for a different city's data format, edit (or create) a city config file:

```json
{
  "city_name": "YourCity",
  "column_mapping": {
    "parcels": {
      "parcel_id": "YOUR_PARCEL_ID_COLUMN",
      "address": "YOUR_ADDRESS_COLUMN",
      "latitude": "YOUR_LAT_COLUMN",
      "longitude": "YOUR_LNG_COLUMN"
    }
  }
}
```

## Bright Data Integration (4 Products)

UrbanPulse integrates **four** Bright Data products to provide comprehensive web
intelligence:

### 1. Web Scraper (Datasets API)

Real-time POI and review data via Google Maps dataset scraping.

| Signal | Source | Description |
|--------|--------|-------------|
| **POI Count** | Google Maps search | Number of nearby Points of Interest |
| **Average Rating** | Google Maps reviews | Mean star rating of nearby businesses |
| **Review Count** | Google Maps reviews | Total number of reviews in the area |
| **Competitor Count** | Google Maps search | Number of businesses in the same category |

**Endpoint**: `POST /datasets/v3/trigger` with dataset `gd_l1vijqt9jfj7olhj`

### 2. SERP API

Search engine results for local business intelligence. Returns structured Google/Bing
results for Montgomery-related queries (e.g., "grocery stores Montgomery AL").

| Feature | Detail |
|---------|--------|
| **Endpoint** | `POST /request` (SERP zone) |
| **Engines** | Google, Bing |
| **Geo-targeting** | Montgomery, Alabama |
| **Output** | Structured JSON with title, URL, description, position |

Used for competitive analysis, market research, and local trend discovery.

### 3. Web Unlocker

Scrapes any public webpage as clean markdown for AI/LLM consumption. Handles
anti-bot protection, CAPTCHAs, browser fingerprinting, and dynamic rendering.

| Feature | Detail |
|---------|--------|
| **Endpoint** | `POST /request` (Unlocker zone) |
| **Output** | Clean Markdown text |
| **Use Case** | Enrich AI copilot answers with live web context |

### 4. MCP Server (Model Context Protocol)

Documented integration for AI agent access to web data via Bright Data's MCP Server.
Supports real-time web data retrieval through tools like `search_engine`,
`scrape_as_markdown`, and `web_data_google_maps_reviews`.

| Feature | Detail |
|---------|--------|
| **Endpoint** | `mcp.brightdata.com/sse` |
| **Free Tier** | 5,000 requests/month |
| **Tool Groups** | Business Intelligence, Social Media, E-commerce, Research |

### API Configuration

| Setting | Environment Variable | Default |
|---------|---------------------|---------|
| API Token | `BRIGHTDATA_API_TOKEN` | (required) |
| Base URL | `BRIGHTDATA_BASE_URL` | `https://api.brightdata.com` |
| SERP Zone | `BRIGHTDATA_SERP_ZONE` | `serp_api1` |
| Unlocker Zone | `BRIGHTDATA_UNLOCKER_ZONE` | `unlocker1` |

### Fallback Mode

When `BRIGHTDATA_API_TOKEN` is not set, all four products automatically switch to
**fallback mode** with deterministic placeholder data, allowing basic demo functionality.
**Note:** ArcGIS, Census, and Weather data remain fully live regardless of Bright Data
configuration — only Bright Data web signals use fallback data when unconfigured.

### Running the Seed Script

```bash
cd backend
python -m scripts.seed_sample
```

This creates (or resets) `backend/urbanpulse.db` with all sample data, ready for
immediate API and UI usage.
