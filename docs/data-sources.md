# UrbanPulse Data Sources

## Overview

UrbanPulse ingests data from three categories of sources:

1. **Montgomery Open Data Portal** — Official city datasets (parcels, incidents, projects)
2. **Bright Data Web Scraper** — POI, review, and competition signals from the web
3. **Seed / Sample Data** — 50 realistic demo properties generated by the seed script

---

## Montgomery Open Data Portal

**Portal URL**: https://opendata.montgomeryal.gov

The portal is powered by ArcGIS and provides REST API access along with downloadable
CSV/GeoJSON files. The ArcGIS base URL used for direct API queries is:

```
https://gis.montgomeryal.gov/arcgis/rest/services
```

### Datasets Used

| Dataset | Description | Target Table | File |
|---------|-------------|-------------|------|
| **Parcels** | Property parcels with zoning, ownership, and coordinates | `properties` | `data/raw/parcels.csv` |
| **Police Incidents** | 911/police incident reports with location and severity | `incidents` | `data/raw/incidents.csv` |
| **City Projects** | Infrastructure and development projects | `city_projects` | `data/raw/city_projects.csv` |
| **Business Licenses** | Active business licenses (used for competition analysis) | — | `data/raw/business_licenses.csv` |

### How to Download Data Files

1. Visit https://opendata.montgomeryal.gov
2. Search for the dataset (e.g., "Parcels", "Police Incidents")
3. Click **Export** → **CSV** (or **GeoJSON** for spatial data)
4. Save the file to `backend/data/raw/` with the expected filename

Alternatively, if ArcGIS REST endpoints are available:

```bash
# Example: download parcels as CSV via ArcGIS REST
curl "https://gis.montgomeryal.gov/arcgis/rest/services/Parcels/FeatureServer/0/query?where=1=1&outFields=*&f=csv" \
  -o backend/data/raw/parcels.csv
```

> **Note**: Actual endpoint paths may vary. Check the portal for current service URLs.

## Column Mapping Configuration

The ingestion script uses a column mapping defined in `config/cities/montgomery.json`
to normalize source CSV columns to the UrbanPulse database schema.

### Parcels Mapping

| UrbanPulse Field | Source Column | Description |
|------------------|--------------|-------------|
| `parcel_id` | `PARCEL_ID` | Unique parcel identifier |
| `address` | `FULL_ADDRESS` | Street address |
| `latitude` | `LATITUDE` | GPS latitude |
| `longitude` | `LONGITUDE` | GPS longitude |
| `property_type` | `PROPERTY_TYPE` | residential, commercial, mixed, vacant_land |
| `zoning` | `ZONING` | Zoning code (R-1, C-2, etc.) |
| `is_vacant` | `VACANT` | Y/YES/TRUE/1 = vacant |
| `lot_size_sqft` | `LOT_SIZE` | Lot area in square feet |
| `assessed_value` | `ASSESSED_VALUE` | Tax-assessed value |
| `year_built` | `YEAR_BUILT` | Year structure was built |
| `neighborhood` | `NEIGHBORHOOD` | Neighborhood name |

### Incidents Mapping

The incidents CSV is expected to have these columns (case-sensitive):

| Column | Description |
|--------|-------------|
| `INCIDENT_ID` | Unique incident identifier |
| `INCIDENT_TYPE` | Category (theft, vandalism, assault, etc.) |
| `LATITUDE` | GPS latitude |
| `LONGITUDE` | GPS longitude |
| `NEIGHBORHOOD` | Neighborhood name |
| `SEVERITY` | low, medium, or high |

### City Projects Mapping

| Column | Description |
|--------|-------------|
| `PROJECT_NAME` | Name of the project |
| `PROJECT_TYPE` | streetscape, infrastructure, park, transit |
| `STATUS` | planned, ongoing, completed |
| `LATITUDE` | GPS latitude |
| `LONGITUDE` | GPS longitude |
| `BUDGET` | Project budget |
| `DESCRIPTION` | Textual description |

### Customizing Mappings

To adapt for a different city's data format, edit (or create) a city config file:

```json
{
  "city_name": "YourCity",
  "column_mapping": {
    "parcels": {
      "parcel_id": "YOUR_PARCEL_ID_COLUMN",
      "address": "YOUR_ADDRESS_COLUMN",
      "latitude": "YOUR_LAT_COLUMN",
      "longitude": "YOUR_LNG_COLUMN"
    }
  }
}
```

## Bright Data Web Scraper

Bright Data provides real-time web intelligence via its Web Scraper API. UrbanPulse
uses it to fetch:

| Signal | Source | Description |
|--------|--------|-------------|
| **POI Count** | Google Maps search | Number of nearby Points of Interest |
| **Average Rating** | Google Maps reviews | Mean star rating of nearby businesses |
| **Review Count** | Google Maps reviews | Total number of reviews in the area |
| **Competitor Count** | Google Maps search | Number of businesses in the same category |

### API Configuration

| Setting | Value |
|---------|-------|
| Base URL | `https://api.brightdata.com` |
| Auth | Bearer token via `BRIGHTDATA_API_TOKEN` env var |
| Dataset | `gd_l1vijqt9jfj7olhj` (Google Maps dataset) |
| Format | JSON |

### Request Flow

1. The client constructs a Google Maps search URL for the target category and coordinates
2. Sends a `POST` to `/datasets/v3/trigger` with the dataset ID and URL
3. Parses the response into POI count, ratings, review counts, and competitor density
4. Computes an **Activity Index** (0–10) from the aggregated signals
5. Caches results in the `web_signals` database table

### Simulated Mode

When `BRIGHTDATA_API_TOKEN` is not set (or set to a placeholder), the client
automatically switches to **simulated mode**. It generates deterministic mock data
using the property's coordinates as a seed, providing:

- POI counts: 3–25 (correlated to latitude/longitude hash)
- Ratings: 2.5–4.8
- Review counts: 10–200
- Competitor counts: 1–8

This allows full demo functionality without a Bright Data subscription.

### Activity Index Formula

```
activity_index = min(10.0,
    (min(poi_count, 50) / 50) × 3
  + (avg_rating / 5) × 3
  + (min(review_count, 500) / 500) × 2
  + (1 - min(competitor_count, 20) / 20) × 2
)
```

## Sample Data

The seed script (`backend/scripts/seed_sample.py`) generates a complete demo dataset:

| Data Type | Count | Description |
|-----------|-------|-------------|
| **Properties** | 50 | Spread across 20 Montgomery neighborhoods |
| **Incidents** | 100 | Random types and severities near properties |
| **Service Locations** | 20 | Groceries, clinics, parks, schools, daycares, libraries |
| **City Projects** | 5 | Infrastructure projects with realistic budgets |
| **Web Signals** | 50 | One signal record per property |
| **Property Scores** | 50 | Pre-computed scores for the "general" scenario |
| **Watchlist Items** | 5 | Sample watchlist entries |

### Running the Seed Script

```bash
cd backend
python -m scripts.seed_sample
```

This creates (or resets) `backend/urbanpulse.db` with all sample data, ready for
immediate API and UI usage.

Woodmere, Ridgewood, Haardt, Zelda Road Corridor, Eastern Boulevard Corridor,
